## 三、显示样式分类（OCR识别特征）

### 样式A：纯数字+单位（最核心）
- 格式："20ms", "45ms", "120ms", "999ms"
- 出现游戏：王者荣耀、和平精英、PUBG、LOL手游、三角洲行动
- OCR特征：数字后紧跟"ms"小写字母，数字通常为2-3位
- 正则建议：`\d{1,3}ms`

### 样式B：图标+数字
- 格式：[WiFi图标]20 或 [信号塔]45
- 出现游戏：王者荣耀(WiFi图标)、和平精英(网络波形图标)
- OCR特征：图标旁有2-3位数字，图标可能干扰OCR
- 建议：先做图标区域检测，再对图标右侧区域做数字识别

### 样式C：FPS+延迟组合
- 格式："FPS 60 | 延迟 25ms" 或 "60FPS 25ms" 或 "FPS:60 Ping:25"
- 出现游戏：三角洲行动、使命召唤手游(开启后)、英雄联盟手游
- OCR特征：同时包含"FPS"和"延迟"/"Ping"字样
- 正则建议：`(?:FPS|fps)[\s:]*\d+.*?(?:延迟|Ping|ping)[\s:]*\d+`

### 样式D：颜色状态指示
- 绿色：< 60ms（良好）
- 黄色/橙色：60-100ms（一般）
- 红色：> 100ms（较差）
- 出现游戏：王者荣耀、和平精英、LOL手游
- OCR建议：结合颜色过滤，优先识别红色/黄色区域

### 样式E：文字提示型（异常状态）
- 格式："网络延迟检测"、"网络异常"、"连接中断"、"网络波动"、"正在重连"
- 出现游戏：PUBG Mobile、火影忍者手游、第五人格、和平精英
- OCR特征：中文字符提示，非数字型
- 关键词库：`["网络延迟", "网络异常", "连接中断", "网络波动", "正在重连", "延迟过高", "网络不稳定"]`

### 样式F：英文提示型
- 格式："Latency: XXms", "Ping: XX", "Network Delay"
- 出现游戏：PUBG Mobile、堡垒之夜、国际服游戏
- OCR特征：英文单词+数字

## 四、OCR脚本提取策略建议

### 1. 区域划分策略
建议将屏幕划分为5个主要检测区域，按优先级排序：

```
优先级1：左上角区域 [X:0%-25%, Y:0%-8%]
  → 王者荣耀、LOL手游、PUBG、三角洲行动、CODM、CFM等

优先级2：右上角区域 [X:75%-100%, Y:0%-8%]
  → 和平精英、香肠派对、堡垒之夜、PUBG Mobile(国际服)等

优先级3：左下角区域 [X:0%-25%, Y:92%-100%]
  → 和平精英(异常状态)、荒野行动(观战)、QQ飞车、棋牌类

优先级4：底部中央区域 [X:35%-65%, Y:92%-100%]
  → 和平精英(训练场/旧版)、棋牌类

优先级5：全屏浮动检测（异常状态）
  → 网络异常弹窗、重连提示、延迟警告
```

### 2. OCR关键词库（优先级排序）
```python
keywords = [
    # 最高优先级：数字+ms
    r"\d{1,3}ms",
    r"\d{1,3} ms",

    # 高优先级：ping相关
    r"[Pp]ing[:\s]*\d+",
    r"[Ll]atency[:\s]*\d+",

    # 中优先级：中文延迟
    r"延迟[:\s]*\d+",
    r"网络[:\s]*\d+",

    # 辅助定位：FPS（常与延迟同时出现）
    r"FPS[:\s]*\d+",
    r"fps[:\s]*\d+",

    # 异常状态
    "网络延迟",
    "网络异常",
    "连接中断",
    "网络波动",
    "正在重连",
    "延迟过高",
]
```

### 3. 颜色过滤建议（HSV色彩空间）

## 五、重要注意事项与边界情况

1. **版本差异**：同一游戏的不同版本（国服/国际服/测试服/体验服）延迟显示位置可能不同
   - 例：PUBG Mobile国际服与国服位置可能不同
   - 例：王者荣耀体验服与正式服UI可能有微调

2. **UI自定义**：部分游戏允许玩家自定义HUD位置
   - 和平精英：可调整部分UI元素位置
   - CODM：HUD高度可自定义
   - 脚本需支持坐标偏移检测

3. **横屏/竖屏差异**：
   - 横屏游戏（绝大多数）：延迟在顶部
   - 竖屏游戏（如棋牌类）：可能在底部或顶部

4. **平板适配**：
   - 平板设备（iPad/安卓平板）UI会重新布局
   - 位置比例需按屏幕百分比而非绝对像素

5. **模拟器差异**：
   - 使用模拟器（腾讯手游助手、雷电、夜神等）时
   - 部分游戏会显示PC端UI布局或键位提示覆盖层

6. **高刷新率手机**：
   - 高刷手机（90Hz/120Hz/144Hz）UI渲染可能与60Hz手机有细微位置差异
   - 但百分比坐标基本保持一致

7. **刘海屏/挖孔屏/灵动岛**：
   - 顶部状态栏被遮挡时，游戏UI可能下移
   - 需检测屏幕安全区域（Safe Area）

8. **游戏内状态变化**：
   - 观战模式 vs 实战模式：位置可能变化（如荒野行动）
   - 大厅 vs 对局内：部分游戏仅在对局内显示
   - 设置菜单打开时：延迟显示可能被覆盖

9. **网络异常状态**：
   - 高延迟时（如999ms），部分游戏会改变显示样式或弹出提示
   - 断线重连时，可能显示"正在连接"而非数字

10. **多语言环境**：
    - 国际服游戏可能显示英文"Latency"或"Ping"
    - 日文/韩文服可能有本地化文字

## 七、总结

对于OCR脚本设计，核心思路是：

1. **优先检测左上角和右上角**：这两处覆盖了80%以上会显示延迟的手游
2. **关键词以"数字+ms"为核心**：这是最常见的延迟显示格式
3. **辅助检测"ping"、"延迟"、"FPS"等关键词**：用于定位延迟显示区域
4. **颜色过滤提高效率**：优先关注红色/黄色区域（高延迟状态）
5. **游戏白名单机制**：对于已知不显示延迟的游戏（如原神、阴阳师等），可直接跳过检测以节省算力
6. **异常状态兜底**：全屏检测网络异常文字提示，覆盖浮动弹窗类显示

## 附录：横屏与竖屏模式适配说明

### 当前手游屏幕模式分布

| 模式 | 占比 | 典型游戏类型 | 延迟显示特点 |
|------|------|------------|------------|
| **横屏** | **~85%** | MOBA、FPS/TPS、大逃杀、ARPG、MMORPG、竞速、格斗 | 延迟显示在**顶部左右角**（左上角或右上角） |
| **竖屏** | **~15%** | 棋牌、消除、音游、跑酷、部分休闲、乙女向 | 延迟显示在**底部**或**顶部中央** |

> 注：竞技类手游（王者荣耀、和平精英、LOL手游等）**全部为横屏模式**，这也是延迟显示需求最强烈的游戏群体。

### 横屏模式下的坐标特征

横屏分辨率常见：1920×1080、2400×1080、2560×1440、2560×1600

```
横屏布局示意：
┌─────────────────────────────────────┐  ← Y[0-8%] 顶部区域
│ [左上角延迟]              [右上角延迟] │     窄条区域，宽度全屏
├─────────────────────────────────────┤
│                                     │
│           游戏主画面区域             │  ← Y[8%-92%]
│                                     │
├─────────────────────────────────────┤  ← Y[92-100%] 底部区域
│ [左下角提示]        [底部中央提示]   │     窄条区域
└─────────────────────────────────────┘
   X[0-25%]                          X[75-100%]
```

**横屏模式下延迟显示规律：**
- 顶部区域高度仅占屏幕的 **5-8%**，但宽度全屏
- 左上角和右上角是**最拥挤的UI区域**，集中了延迟、FPS、信号、电池、时间等信息
- 底部区域高度同样仅占 **5-8%**
- 主画面区域（Y[8%-92%]）**几乎不会出现延迟数字**

### 竖屏模式下的坐标特征

竖屏分辨率常见：1080×1920、1080×2400、1170×2532

```
竖屏布局示意：
┌─────────────────┐  ← Y[0-5%] 顶部状态栏
│    时间/信号     │
├─────────────────┤  ← Y[5-10%] 游戏顶部UI
│  [顶部中央延迟]  │     部分竖屏游戏在此显示
├─────────────────┤
│                 │
│   游戏主画面     │  ← Y[10%-85%]
│                 │
├─────────────────┤  ← Y[85-95%] 底部操作区
│ [底部延迟/网络]  │
├─────────────────┤  ← Y[95-100%] 底部导航
│                 │
└─────────────────┘
   X[0-50%]  X[50-100%]
```

**竖屏模式下延迟显示规律：**
- 顶部状态栏（系统级）可能显示网络信号，但**非游戏内延迟**
- 游戏内延迟（如有）通常出现在：
  - **底部操作区附近**（如棋牌类）
  - **顶部UI区域**（如部分跑酷游戏）
- 竖屏游戏的延迟显示**远不如横屏游戏普遍**

### 本方案对横屏/竖屏的兼容性

由于本方案使用**百分比坐标**（而非绝对像素），因此对横屏和竖屏均适用：

| 检测区域 | 横屏适配 | 竖屏适配 | 说明 |
|---------|---------|---------|------|
| 左上角 [0-25%, 0-8%] | ✅ | ✅ | 横屏左上角 / 竖屏左上角 |
| 右上角 [75-100%, 0-8%] | ✅ | ✅ | 横屏右上角 / 竖屏右上角 |
| 左下角 [0-25%, 92-100%] | ✅ | ✅ | 横屏左下角 / 竖屏左下角 |
| 底部中央 [35-65%, 92-100%] | ✅ | ✅ | 横屏底部中央 / 竖屏底部中央 |
| 顶部中央 [35-65%, 0-8%] | ✅ | ✅ | 横屏顶部中央 / 竖屏顶部中央 |

**横屏优化建议：**
- 横屏游戏顶部区域较窄（Y[0-8%]），OCR识别区域可以**更精确**
- 建议横屏模式下将顶部检测区域细分为：
  - 左上角精确区：[0-20%, 0-6%]
  - 右上角精确区：[80-100%, 0-6%]
- 这样可以减少主画面区域的误识别

**竖屏补充建议：**
- 竖屏棋牌类游戏（欢乐斗地主、麻将）延迟显示在底部
- 竖屏游戏底部区域相对更高（Y[85-100%]），可适当扩大检测高度
- 建议竖屏模式下底部检测区域调整为：[0-100%, 85-100%]

脚本中可添加自动判断逻辑：

    h, w = image.shape[:2]
    return w > h  # 宽度大于高度为横屏

def get_adaptive_regions(image, game_name=None):

            "top_center": (0.30, 0.0, 0.70, 0.08),  # 竖屏顶部中央
